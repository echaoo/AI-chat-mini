## 首页无固定角色时，把和用户最后一次聊天的角色自动固定到首页，
### 实现思路（基于当前代码结构）：

  - 数据来源：src/components/character-card-content.mpx 在组件卸载时会写入 last_conversation_character（最近对话角色）和 last_conversation_id，这就是“最后一次聊天”的
    本地线索。
  - 触发点：首页 onShow -> loadHomeCharacter -> syncHomeCharacterFromServer。当服务端返回“无固定角色”时，尝试读取 last_conversation_character。
  - 自动固定流程：
      1. 若本地 last_conversation_character 存在且字段完整，先用它更新首页 UI 和 home_character 缓存（保证立刻展示）。
      2. 同时调用后端 pin 接口（characterApi.togglePinToHome(characterId)）把它同步固定到服务端。
      3. 若 pin 失败或缓存无效，再清理 home_character 并走原有“跳到角色选择”的兜底逻辑。
  - 兜底与一致性：若没有本地 lastConversation 或接口失败，就保持当前逻辑：清缓存并引导用户去角色选择页。
